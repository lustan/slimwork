# 处理评审反馈

在修改前先理解反馈意图，避免机械照改。

## 处理方式

1. 逐条分类：必须修复 / 建议优化 / 需讨论
2. 对不清晰项先澄清上下文与目标
3. 对技术争议项给出证据（规范、测试、性能）
4. 修改后逐条回复并附验证结果

## 注意事项

- 不做“看起来合理但未经验证”的改动
- 不遗漏评审线程闭环
