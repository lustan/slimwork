# 开发分支收尾

实现完成后，系统化完成收尾、合并与交付说明。

## 核对清单

- 需求范围已完成
- 相关测试与检查通过
- 无无关改动、临时代码、调试残留
- 提交历史清晰可读

## 集成建议

1. 选择合适集成策略（merge / squash / rebase）
2. 补齐 PR 说明：动机、改动、风险、验证
3. 标注潜在回滚点与兼容影响
4. 完成最终审阅后合并
