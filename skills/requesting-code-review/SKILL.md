# 发起代码评审

在关键里程碑发起评审，提升质量并降低合并风险。

## 发起前准备

- 说明目标、范围与非目标
- 归纳关键设计决策与风险点
- 提供可复现的验证命令
- 标明希望重点关注的区域

## 跟进方式

1. 及时回应反馈
2. 变更后补充对比与验证
3. 确认所有线程已解决
